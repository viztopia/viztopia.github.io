YORB in Action

YORB in Action (temporary name) explores the use of the virtual 3D space Yorb for live performance, on top of its current event hosting features. It comprises of a series of experiments that include building tools for viewing, creating & managing 3D assets / space / scenes, sequencing scenes / events, managing participant feeds, and making novice interactions possible.

Below is a tentative list of components that will be investigated / developed in the upcoming months.

YORB Viewer


YORB Viewer is a tool for designers to quickly preview the layout of assets in the space, and make adjustments before the assets are imported into the space. It loads the ITP floor by default and you can drag and drop assets into it for preview. Check it out here.

Assets Manager



Assets Manager is a tool for space / stage designers to add / edit / remove 3D assets for the designated YORB space. Currently it's connecting the frontend YORB client to a backend Firebase, and it can potentially be upgraded to a YORB Editor.

Performer & Audience Controller


Performer & Audience Controler is a tool that is built on top of YORB's current WebRTC MediaSoup APIs. It provides a higher layer of control over the visibility / loading of participant streams, so that we can control who will be the performers and show their streams on stage, and who will be merely the participants with no streams. A mock up of a performer-only rainy night scene can be found here.
